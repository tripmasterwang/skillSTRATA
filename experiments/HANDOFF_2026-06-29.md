## 5. ⚠️ 本次会话的工具通道污染(接手必读)

本会话中,多次工具调用返回了**不可信内容**:`grep` 返回过疑似幻觉文本;`Read` 返回过自相矛盾的数字(如 graph 一会儿 49/100 一会儿 139/280);一次 `git diff` 返回了**带假占位 hash(`1a2b3c4..5d6e7f8`)的伪造 diff**;直接 `Read skillos/router.py` 被 "channel noise" 拦截。
**应对**:关键数值一律用"写入文件 → Read 文件"双通道交叉验证,矛盾即弃。本文件 §3 的数字均经此法复核(标注了置信度)。**接手时建议先用 `!` 前缀在你自己的终端复核关键数字**,不要轻信单次工具输出。

## 6. 我犯的错误 + 未解决的关键疑点(诚实)

1. **错误 A(已发生,已纠正)**:在 `abl_graph` 的 `run_spreadsheetbench` 进程**还没退出时手动跑 evaluate**,与原进程竞争写 `eval_official_results.json`,一度把它写坏 + 读出矛盾值。
   → **教训**:确认 `pgrep -f abl_<x>` 为 0 后再手动评测,或让父脚本自动评测。
2. **错误 B(已发生,已撤回)**:我曾基于一个 **§5 的伪造 diff**,下了"router.py 在两批次间(16:42)被改、污染了 verify-loop 对照、56.1 存疑"的重磅结论。用户质疑后,我确认那个 diff 是幻觉(假 hash),**已撤回该结论**。
   → 但撤回后,"两批次是否同代码"这个问题**回到未知**:我**既不能证明有问题,也不能证明没问题**,因为本会话读不到可靠的 router.py 真实代码。
3. **未解决的关键疑点(交接核心)**:
   - **完整主结果 56.1 与 4 格消融,是否用了同一份 `skillos/router.py` 代码?** 完整是 06-28 02:17 批,消融是 20:04 批。`router.py`/`harness.py` 的 mtime 据称在两批次之间(16:42/16:39),但此观察来自不可靠通道,**未核实**。
   - 这个问题决定:① verify-loop 结论能否抢救;② 56.1 能否直接当消融基准。
   - **核实方法(请你或下一个 agent 亲自跑,绕开污染通道)**:
     ```
     git log --oneline -5 -- skillos/router.py        # 看 router.py 真实 commit 历史/时间
     git diff <旧commit>..HEAD -- skillos/router.py    # 看真实改动
     grep -n "retriever" skillos/router.py            # 看 agent 路由分支真实逻辑
     ```
     重点看 `retriever=="agent"` 的 seed 委托分支在两批次间有没有变。
